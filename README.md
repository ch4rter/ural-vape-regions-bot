# Telegram-бот «Регион — менеджер»

Бот ищет введённое название в Excel-файле и показывает закреплённого менеджера. Он не определяет географию самостоятельно: каждое нужное название должно присутствовать в таблице.

Также бот содержит раздел `🗃 База данных` с товарами, разделами и рабочими материалами. Управлять им может только Telegram-пользователь, чей ID указан в `ADMIN_IDS`.

Раздел `💰 Цены и наличие` ищет товарные группы по складским прайсам, показывает ассортимент по Москве, Санкт-Петербургу и Челябинску и рассчитывает цены со скидками 5%, 10% и 15%.

Из результатов поиска можно добавить одну или сразу все найденные группы в подборку и сформировать отдельный клиентский Excel-прайс. Для каждого склада бот делает отфильтрованную копию исходного прайса: сохраняет его оформление, формулы, объединённые ячейки и изображения, удаляя лишние товарные строки. В колонках `I–K` (`Москва`, `Санкт-Петербург`, `Челябинск`) для каждого вкуса или цвета наличие отмечается символами `✅` и `❌`; строка с названиями колонок закреплена в верхней части окна. Доступны базовые цены и вариант со скидкой 10%. Подборка рассчитана максимум на 50 товарных групп.

Доступ к боту ограничен белым списком. Администратор добавляет сотрудников по Telegram ID или `@username` через `⚙️ Управление базой → 👥 Белый список`. При первом входе разрешённый username привязывается к постоянному Telegram ID.

## Быстрый запуск в Windows

1. Установите Python 3.10 или новее.
2. Создайте бота через Telegram-аккаунт `@BotFather` и скопируйте токен.
3. Откройте PowerShell в папке проекта и выполните:

```powershell
python -m venv .venv
.venv\Scripts\python -m pip install -r requirements.txt
.venv\Scripts\python create_template.py
Copy-Item .env.example .env
```

4. Откройте `.env` и замените значение `BOT_TOKEN` на токен от BotFather.
5. Запустите:

```powershell
.venv\Scripts\python bot.py
```

Остановить бота можно сочетанием `Ctrl+C`.

## Excel-файл

По умолчанию бот читает `managers.xlsx`, первый лист и первые две колонки:

| Название | Менеджер |
|---|---|
| Тамбов | Андрей |
| Тамбовская область | Андрей |
| Минск | Елена |

- Заголовок допустим, но не обязателен.
- Пустые строки пропускаются.
- Поиск не зависит от регистра, повторных пробелов и различия `е/ё`.
- Одно название можно записать несколько раз. Бот покажет всех соответствующих менеджеров.
- После изменения Excel перезапустите бота, чтобы загрузить новые данные.
- Не держите Excel открытым во время замены файла на сервере.

Путь к другой таблице можно задать параметром `EXCEL_PATH` в `.env`.

## База данных товаров

В главном меню доступны два независимых режима:

- `🔎 Найти менеджера` — после включения бот принимает текстовые запросы по территориям;
- `🗃 База данных` — каталог товаров и подготовленных материалов.

Администратору дополнительно доступна кнопка `⚙️ Управление базой`. Через неё можно создавать, переименовывать, скрывать и удалять товары, создавать разделы и добавлять в них текст, изображения и документы.

Сами файлы повторно не сохраняются на сервере: база хранит выданные Telegram идентификаторы файлов. Структура каталога и тексты находятся в `data/materials.sqlite3`. Этот файл не загружается в Git и должен резервироваться отдельно.

Настройки в `.env`:

```env
MATERIALS_DB=data/materials.sqlite3
MANAGED_EXCEL_PATH=data/managers.xlsx
PRICES_DB=data/prices.sqlite3
PRICE_STORAGE=data/prices
ADMIN_IDS=5533726476
```

Несколько администраторов можно указать через запятую.

Администратор может скачать ZIP-резервную копию прямо в Telegram. В архив входят база товаров `materials.sqlite3`, действующая таблица `managers.xlsx` и краткое описание.

## Рассылки по клиентским чатам

Бот автоматически сохраняет название и Chat ID групп, в которые его добавляют после установки этой версии, и отмечает чат неактивным при удалении бота. В разделе `📣 Рассылки клиентам` администратор рассылок может выгрузить реестр в Excel, создать новую рассылку и загрузить отдельный `.xlsx` с обязательной колонкой `Chat ID`.

Рассылка поддерживает один готовый Telegram-пост: текст, фото, видео, анимацию или документ с подписью. Бот показывает предпросмотр, обязательно копирует пост в выбранную служебную группу и только после подтверждения начинает массовую отправку. Дубли Chat ID удаляются, ход отправки показывается в Telegram, а после завершения формируется Excel с результатом по каждому чату.

В белом списке доступны роли `Пользователь` и `Младший администратор`. Младший администратор получает доступ к рассылкам, но не к управлению товарами, прайсами, резервными копиями и ролями. Назначать роли может только главный администратор из `ADMIN_IDS`. Тестовый пост всегда отправляется в служебный чат `-5565597780`.

Telegram не предоставляет боту готовый список групп, куда он был добавлен до установки функции. Такие старые группы нужно один раз переподключить: удалить бота и добавить снова. В клиентских группах бот работает молча и не отвечает участникам.

Новый Excel также можно загрузить через административную панель. Бот проверяет обязательные колонки, показывает число корректных строк и применяет файл только после подтверждения. Загруженная таблица сохраняется в `data/managers.xlsx`, поэтому обновления кода через Git её не перезаписывают. Предыдущие версии автоматически сохраняются в `data/backups/`.

## Цены и наличие

Администратор загружает отдельный `.xlsx` для каждого склада через `⚙️ Управление базой → 💰 Обновить прайсы`. Перед применением бот показывает дату, число групп, число товарных позиций и количество исключённых акционных строк. Обновление одного склада не затрагивает остальные.

При повторной загрузке склада бот сравнивает новый прайс с предыдущим по кодам товаров и формирует отчёт: появившиеся и закончившиеся позиции, изменения цен, новые и исчезнувшие товарные группы. Для каждого склада хранится только последнее сравнение. Краткая сводка и подробный Excel доступны всем разрешённым пользователям в разделе `📊 Изменения прайсов`.

После формирования сегодняшних отчётов Москвы, Санкт-Петербурга и Челябинска администратор может вручную нажать `📣 Отправить общий отчёт`. Бот объединяет изменения трёх складов в одно уведомление, показывает предпросмотр и количество получателей, а отправка начинается только после отдельного подтверждения. Один и тот же набор отчётов нельзя разослать повторно. Уведомление получают пользователи белого списка с привязанным Telegram ID — то есть те, кто уже хотя бы один раз обратился к боту.

Товарные группы определяются по строкам прайса без цен. Для показа берётся последняя часть пути после `/`, а полный путь участвует в умном поиске. Следующие строки с ценами считаются вариантами группы до следующего раздела. У акционных позиций убирается только служебная приписка `АКЦИЯ`; сами товары, цены и наличие сохраняются. Акционные строки из общего раздела распродажи автоматически объединяются в группы по названию модели.

Исходные прайсы сохраняются в `data/prices/`, предыдущие версии и копии базы — в `data/prices/backups/`. Эти рабочие данные исключены из Git.

В пользовательской вкладке `📄 Прайсы` можно скачать исходный файл выбранного склада либо автоматически сформированную копию со скидкой 10%. В копии пересчитываются обе базовые колонки нал/безнал, заголовки получают отметку о скидке, а исходный файл остаётся неизменным.

## Перенос на сервер

Скопируйте папку проекта и `managers.xlsx` на сервер, установите зависимости, создайте `.env` с токеном и запустите `python bot.py`. Для постоянной работы процесс позднее можно оформить как службу `systemd` или запустить в Docker.
